Contributing to Dissonance Docs

Guidelines
- Keep content factual and cite sources where relevant.
- Use Markdown for all docs.
- Add new files in the appropriate folder and update README.md if you introduce new top-level sections.
- For large changes, open a draft PR and assign a reviewer.

Style
- Prefer concise headings and short paragraphs.
- Use consistent date format: YYYY-MM-DD.

PR checklist
- Title: short descriptive title.
- Body: what changed and why.
- Attachments: add images or files to /assets and reference them.
